#### 6. `http.go` — 日志前置后置的模式浪费了一次 `Infow` 调用

`LogRequest` 中间件用 `Infow` 记录 HTTP 日志，但 `[HTTP LOG]` 作为 msg，所有信息都放在字段里，msg 本身几乎是个常量。每条 HTTP 请求产生一条结构化日志，字段中 `method`、`path`、`status`、`duration`、`remote_addr`、`user_agent`、`content_len` 共 7 个字段，在 JSON 格式下这 7 个 key 每条请求都重复序列化。

**建议**：不算严重问题，高流量 HTTP 服务可考虑优化为使用 `Def` 格式直接拼字符串减少字段开销，或者提供配置让用户选择格式。

### ⚠️ 设计权衡（可优化但非缺陷）

#### 7. `logger.go` — `log()` 热路径中的 `append` 分配

```go
entry.Fields = append(entry.Fields[:0], l.config.Fields...)
entry.Fields = append(entry.Fields, fields...)
```

当 `Config.Fields` 不为空时，这两行可能会因为底层 cap 不足而触发二次分配。EntryPool 创建时 `Fields: make([]Field, 0, 8)`，如果 `len(config.Fields) + len(fields) > 8`，就会发生 realloc。

**建议**：可以在 GetEntry 时或 EntryPool.New 时考虑更大的初始 cap，或者在 log() 中预计算总长度。

#### 8. `sampler.go` — `Allow()` 中的 `time.Now()` 调用

每次采样判断都调用 `time.Now()` 获取当前时间戳。在非采样场景（`sampler == nil`）下不会走到这里，但启用采样后每条日志都会调用一次 `time.Now()`。

`time.Now()` 开销约几十纳秒，和日志写入的 IO 开销相比微不足道，**当前不需要优化**，但值得标记为感知点。

#### 9. `sampler.go` — `Allow()` 重置窗口时的竞态窗口

```go
if tn > c.resetAt.Load() {
    c.counter.Store(1)
    c.resetAt.Store(tn + s.tick.Nanoseconds())
    return true
}
```

当窗口过期时，多个 goroutine 可能同时检测到 `tn > resetAt`，各自重置计数器并 `return true`。这导致窗口切换时**可能多发几条日志**，但不会丢失日志，是 zap 也采用的"宽松重置"策略。

**这不算 bug，算已知行为**，如果需要严格重置需要 CAS 循环，会增加热路径开销。

#### 10. `logger.go` — Entry 池中 `TimeFormat` 未清理

```go
func PutEntry(e *Entry) {
    e.Fields = e.Fields[:0]
    e.Caller = ""
    e.Message = ""
    e.Time = time.Time{}
    EntryPool.Put(e)
}
```

`TimeFormat` 没被 reset，但因为它每次在 `log()` 里都会被直接覆盖赋值：

```go
entry.TimeFormat = l.config.TimeFormat
```

所以不清理是安全的（之前讨论过）。但如果以后在 Entry 使用过程中改变了其值且未覆盖，可能有问题。这属于**长期维护风险点**，建议加一行清理让行为一致。

### 🧹 代码风格/微小建议

| #    | 位置               | 问题                                                         | 建议                                                     |
| ---- | ------------------ | ------------------------------------------------------------ | -------------------------------------------------------- |
| 11   | `config.go:71`     | `Fields: []Field{}` 而非 `nil`                               | `nil` 和空切片行为一样，改 `nil` 可避免一次分配          |
| 12   | `config.go`        | `SamplerInitial`/`SamplerThereafter` 注释用了"放行"口语      | 和 `SamplerTick` 的注释风格不统一（tick 用了"一个窗口"） |
| 13   | `field.go:Stack()` | `Stack()` 返回 `StringType`（字段值是 `debug.Stack()` 字节转 string） | 大堆栈会直接拼接到日志中，若日志量大会膨胀内存           |
| 14   | `http.go`          | `LogRequest` 参数类型为 `*Logger` 不是 `*Logger` 接口        | 无法传入 mock 做单元测试                                 |
